# TruthSeeker 本地启动方式

> 更新时间：2026-06-06。完整项目介绍和环境变量分类见 `README.md` 与 `docs/TECH_STACK.md`。

## 前置条件

- Node.js 20+
- npm
- Python 3.11+
- 已准备 Supabase 项目
- 按需准备 Agent LLM、媒体检测、威胁情报、域名溯源和 embedding 服务 key

不要把真实密钥写进文档或提交到 Git。

## 1. 后端

> 新建 Supabase 项目时，当前仓库迁移尚不包含上传接口所需的 `media` Storage bucket 与 Storage RLS。需先在 Supabase 中配置，或完成 `docs/KNOWN_GAPS.md` 中对应迁移任务，否则上传会失败。

首次运行：

```powershell
cd truthseeker-api
Copy-Item .env.example .env

# 当前 Windows 工作区优先复用 venv_new；不存在时再创建
python -m venv venv_new
.\venv_new\Scripts\python.exe -m pip install -r requirements.txt
.\venv_new\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
```

日常启动：

```powershell
cd truthseeker-api
.\venv_new\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
```

验证：

```powershell
Invoke-RestMethod http://localhost:8000/health
```

Swagger UI：`http://localhost:8000/docs`

## 2. 前端

首次运行：

```powershell
cd truthseeker-web
Copy-Item .env.example .env.local
npm install
npm run dev
```

日常启动：

```powershell
cd truthseeker-web
npm run dev
```

前端地址：`http://localhost:3000`

## 3. 环境变量

### `truthseeker-web/.env.local`

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=sb_publishable_...
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

### `truthseeker-api/.env`

从 `.env.example` 复制后填写。至少需要确认：

- Supabase：`SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY`、`SUPABASE_ANON_KEY`、`SUPABASE_JWT_SECRET`
- Agent LLM：`AGENT_LLM_PROVIDER` 与对应的 `KIMI_*` 或 `MIMO_*`
- 可选工具：`SIGHTENGINE_*`、`REALITY_DEFENDER_API_KEY`、`VIRUSTOTAL_API_KEY`、`EXA_API_KEY`、`WHOISXML_API_KEY`
- RAG：`EMBEDDING_*`

生产环境不能使用 `SUPABASE_JWT_SECRET=NOT_SET`。

## 4. 检查命令

后端：

```powershell
cd truthseeker-api
.\venv_new\Scripts\python.exe -m pytest tests
```

前端：

```powershell
cd truthseeker-web
npm run lint
npm run typecheck
npm run test:unit
npm run build
```

真实端到端检测还需要 Supabase、Storage、外部 API key 和前后端同时运行。
