# TruthSeeker - 基于鲁棒联邦学习与多智能体交叉验证的跨模态 Deepfake 鉴伪与溯源系统全面技术与项目白皮书

## 1. 行业背景与核心痛点分析

### 1.1 背景：生成式 AI 技术爆炸与“后真相时代”

在 2025-2026 年，生成式 AI（AIGC）技术迎来了现象级爆发期。大语言模型（LLM）、扩散模型（Sora、Stable Diffusion）和语音合成技术（VALL-E、ElevenLabs 等）极大地降低了内容制作门槛。然而，这种低门槛直接催生了新型网络安全威胁：

- **Deepfake 换脸诈骗**：不法分子利用 AI 实时替换视频中的面部信息和语音，伪装成公司高管、公职人员或受害者亲友，实施精准定向金融诈骗。
- **政治与社会虚假信息操纵**：利用“深度伪造”技术生成政治人物的虚假演讲视频，或在突发事件中炮制极其逼真的伪造图片/视频，严重扰乱社会公共秩序。
- **跨模态复合型欺诈**：现代网络谣言已从单纯的“假文字”演变为“假新闻稿 + 假视频证据 + 恶意钓鱼链接 + 虚假时间戳”的复合体。

### 1.2 现有鉴伪方案的“致命局限”

针对上述威胁，传统的单点反制方案显然已经捉襟见肘，主要体现在以下五个层面：

1. **数据孤岛与隐私壁垒（阻碍模型进化）**：高质量的伪造样本往往散布在各大平台、公安边缘节点或个人设备中，受限于数据隐私法规，机构间无法共享数据进行集中训练，导致检测模型难以覆盖最新变种。
2. “猫鼠游戏”下的单模型脆弱性**：现有工具多采用单一深度学习分类器（如单纯检测视频的卷积网络）。当攻击者使用最新的生成技术（如对抗生成网络 GAN 添加无法察觉的噪声）时，单模型极易失效，难以应对 Zero-Day（零日）伪造技术。
3. **“模态孤岛”导致关联缺失**：一段由 AI 合成的演讲视频，其视频画面、音频频谱、甚至新闻传播链接往往存在内在的逻辑矛盾（例如：视频口型与发音细微不匹配；新闻来源是一个注册仅 3 天的恶意域名）。现有工具只能“看图识图、听音辨音”，缺乏将跨模态特征进行联合验证（Cross-Validation）的全局智能。
4. **“黑盒判定”不可查证**：单纯输出一个“置信度 99% 为假”的数字无法成为具有法律效力的依据，缺乏具体的“数字法医”推理过程（例如指出哪一帧边缘光影不自然，或哪段音频频谱被截断）。
5. **人工研判效率低下**：在面对海量疑似信息时，专业安全人员无法逐帧分析，平台亟需一个高吞吐量、自动“辩论”找茬的辅助研判中枢。

## 2. 核心架构与差异化定位：感知与推理的解耦双擎架构

### 2.1 项目定位

TruthSeeker 不是另一个简单的 Deepfake 过滤插件，而是一个**融合了分布式联邦隐私计算与“数字法医”思维的智能体研判中枢**。它模拟了现实世界中专案组的运作模式：专家各司其职、交叉质询、收敛裁定。

针对上述痛点，本项目将感知与推理解耦，开创性地提出了**“底层感知 + 上层推理”的双层协同架构**：

- **底层感知基座（防投毒隐私联邦学习）**：采用 PARSIFAL 核心技术架构，在保护各边缘节点数据隐私的前提下，联合训练出极具鲁棒性的专精检测模型（视觉伪造检测、音频异常检测、恶意 URL 识别），并将其封装为底层 API。
- **上层推理大脑（多智能体交叉验证）**：利用大模型作为“逻辑引擎”，调度多个特化智能体（Agent）调用底层的检测 API。通过模拟现实世界专案组“专家各司其职、交叉质询、收敛裁定”的模式，将黑盒的感知信号转化为白盒的、具有高度可解释性的研判逻辑链。

### 2.2 TruthSeeker 全景技术拓扑

系统采用密码学架构与前沿的 Serverless 全栈技术融合构建：前端（Next.js 3D 沉浸式空间） + 后端（基于 LangGraph 的多智能体流） + 数据与实时通信层（Supabase & WebSocket）。

**核心技术栈：**

- **底层感知基座**：基于 PARSIFAL (Private and Robust Sign Federated Learning) 架构，结合秘密共享（Secret Sharing）与草图（Sketch）机制，实现抗投毒的联邦学习引擎。
- **前端重构级现代化**：基于 Next.js 15 和 React 19，完全抛弃笨重的传统 CSS 配置，采用 Tailwind CSS v4 (CSS-first) 和 `motion v12`。
- **3D 可视化引擎**：引入 `@react-three/fiber` (v9.5+) 构建数字法医工作台（Bento Box 布局），实现液体玻璃高阶着色器特效。
- **后端调度大脑**：基于 FastAPI，并深度集成 LangGraph v1.0+ 构建状态机，利用 `TypedDict` 进行高稳定的上下文流转。
- **Serverless 数据库与实时中枢**：基于 Supabase 构建 PostgreSQL 数据库，并重度利用 Supabase Realtime (Broadcast & Presence) 完成 Agent 状态分发与人类专家在线协作。

## 3. 核心创新一：底层技术基座：基于 PARSIFAL 的隐私且鲁棒的联邦学习感知底座

为了在不泄露各节点敏感原始数据与防御恶意节点投毒攻击的前提下，训练出强大的多模态特征“感知”底座，系统底层采用了 **PARSIFAL（Private and Robust Sign Federated Learning）核心技术架构**。该底座专门用于训练并向大模型多智能体层提供抗投毒的专业检测 API（如：视觉伪造检测、音频异常检测、恶意 URL 识别）。

PARSIFAL 的完整训练与防护流程综合利用了密码学中的秘密共享（Secret Sharing）技术和机器学习中的草图（Sketch）数据结构，分为四个主要阶段：

### 3.1 客户端本地训练与数据加密 (Local Training)

为了减少通信量并为后续的加密验证做准备，客户端不直接上传完整的浮点数梯度。

- **旋转与取符号**：客户端计算出本地梯度后，首先对其应用一个公开的随机旋转变换（Rotation Transform），然后提取其符号（正数为1，负数为0），得到“符号梯度” $q_{i}=Sign(Rot(g_{i}))$。引入旋转操作可以显著降低符号化带来的量化误差，并更好地保留梯度的方向特征。
- **构建草图 (Sketching)**：客户端根据其符号梯度生成一个特殊矩阵形式的“草图”数组。这个草图将用于后续在密文状态下计算梯度之间的相似度。
- **秘密共享**：符号梯度通过二进制秘密共享（BSS）进行加密，而草图通过算术秘密共享（ASS）进行加密，随后分别发送给两个不串通的服务器。

### 3.2 密文状态下的上传验证 (Upload Verification)

由于恶意客户端可能会上传不符合规则的伪造密文数据以破坏密码学协议，服务器必须在不知道明文的情况下验证数据的合法性。

- **梯度验证**：服务器只需低成本地检查 BSS 共享的符号梯度是否为合法的二进制向量。
- **草图验证**：服务器利用同余方程，验证草图矩阵的每一行是否严格为独热（one-hot）向量或全零向量。
- **一致性验证**：为防止客户端对梯度和草图分别造假，服务器使用不经意传输（Oblivious Transfer, OT）技术执行安全的草图查询协议，验证客户端上传的草图与其符号梯度是否逻辑一致。

### 3.3 基于草图的恶意梯度检测 (Poisoning Gradient Detection)

该模块用于揪出那些企图破坏模型的投毒梯度。

- **草图合并**：服务器将所有通过上述验证的本地草图合并为一个“全局草图”。这个全局草图代表了当前轮次所有合法梯度的整体分布情况。
- **密度估计与过滤**：服务器在密文下利用每个客户端的符号梯度去查询这个全局草图，以此估算出该梯度的“密度”。密度代表了该梯度与绝大多数梯度的相似程度。那些密度低于平均阈值的梯度会被判定为离群值（即潜在的投毒梯度），从而被剔除。

### 3.4 基于符号的鲁棒聚合 (Robust Aggregation)

- **中位数聚合**：对于最终通过密度验证的符号梯度，服务器会在密文状态下计算它们的逐坐标中位数（Coordinate-wise Median）。中位数规则本身就具备极强的抗异常值干扰能力。
- **限制梯度长度**：采用符号梯度进行聚合，从根本上限制了任何单一梯度的幅度大小，直接消除了攻击者通过放大梯度权重来实施“模型替换攻击”的可能。
- **更新模型**：最终聚合出的单一符号梯度被解密并发送给模型所有者，模型所有者对其进行逆旋转变换后更新全局模型。这些经过多轮安全聚合迭代得到的精准鉴伪模型，将被封装为核心 API，无缝供给上层大模型智能体调用。

## 4. 核心创新二：上层推理大脑：基于 LangGraph 的多智能体动态辩论系统

TruthSeeker 摒弃了落后的“单模型流水线”检测，并基于底层 PARSIFAL 训练出的高鲁棒 API，首创了基于有向无环图（DAG）和收敛算法的动态辩论状态机。在这个系统中，四个高度特化的 Agent 会针对同一个案件在大脑深处展开“辩论”。全局状态（Global State）由严密的 `InvestigationState` 结构定义，内置包含视觉、音频、情报等独立证据池的电子证据板（Evidence Board）。

### 4.1 四大“特工”职责设定

1. **🎬 视听鉴伪Agent Agent (Forensics Agent)**
   - **行为特征**：充当“物证鉴定中心”。负责调用底层联邦学习底座提供的**视觉伪造检测**和**音频异常检测**模型 API，以及底层的视频抽帧逻辑 (FFmpeg)。它输出纯事实数据：比如“在第 15 秒到 18 秒，检测到面部边缘融合伪影，置信度 85%”。
2. **🔍 情报溯源Agent Agent (OSINT Agent)**
   - **行为特征**：充当“背景调查员”。调用底层联邦学习训练出的**恶意 URL 识别模型**，并集成 VirusTotal API、WHOIS 解析与网页内容爬虫。分析输入数据中携带的元数据、链接或 IP 来源。如果视频看起来完美，但发布源域名被联邦底座识别为高危黑名单，它会在“证据板”上钉上危险警告。
3. **⚖️ 逻辑质询Agent Agent (Challenger Agent - 核心破局点)**
   - **行为特征**：充当“交叉询问律师”。该 Agent 不直接检测，而是审视证据板上的线索进行逻辑链检验。
   - **动态打回机制**：如果发现矛盾（法医说视频是真的，情报犬说发布者是高危诈骗源）或证据置信度低于阈值（< 0.7），它会生成包含目标 Agent、目标证据 ID 和所需行动的 `Challenge` 对象。通过状态机引擎的条件边 (Conditional Edge) 将案卷打回对应 Agent 要求重审（例如：“请换用精度更高的音频声纹 API 再查最后五秒”）。
4. **👨‍⚖️ 研判指挥Agent Agent (Commander Agent)**
   - **行为特征**：充当“法官”。当辩论达到“收敛条件”时接管控制权。对所有带权重的证据连进行综合研判，将底层的机器感知信号转化为具有强解释性的“法医级最终调查报告”。

### 4.2 收敛算法与动态停止机制 (Dynamic Convergence)

在传统的多 Agent 系统中，Agent 容易陷入无休止的“抬杠”死循环。TruthSeeker 在 LangGraph 中设计了严密的量化状态评估与容错收敛算法：

- **权重变化追踪**：系统会在全局 State 中记录每轮辩论后各个 Agent 的置信度得分 (`confidenceHistory`)。
- **收敛判定条件**：
  1. **梯度收敛**：计算当前轮次与上一轮次各模态评估结果的差值向量 ($\Delta$)。如果连续两轮结果变化量 $\Delta < 0.05$（即 5% 的阈值），证明共识已经形成。
  2. **逻辑闭环**：质询官 Agent 的满意度 (`satisfaction`) 大于 0.8。
  3. **最小轮次保障**：系统至少完成 2 轮以上的交叉辩论。
  4. **兜底防御策略 (Fallback)**：为了应对极具对抗性的案例导致结果震荡，设计了最大辩论轮数锁 (`maxRounds` 默认 5 轮）。一旦触发该硬性阈值，无条件终止辩论，进入指挥官强行宣判节点，保护服务器算力。

## 5. 人机协同：专家会诊与 WebSocket 实施通信

为了填补“算法死角”，TruthSeeker 构建了一套实时人机协作房间，将办案流程从全自动升维到“专家监督学习”。

### 5.1 事件驱动流 (Event-Driven Stream)

当案件正在被四个 Agent 处理时，用户的界面需要呈现如谍战片中“代码正在入侵”的即时反馈。后端 FastAPI 构建了原生 SSE（Server-Sent Events）接口。后端通过 `astream` 获取工作流节点的每次状态变化，通过生成器 (Generator) 逐条下发给前端打字机引擎，规避了短连接轮询造成的庞大开销。

### 5.2 Supabase Realtime 协作体系

当 `Challenger Agent` 连续三次对某段核心证据产生疑惑抛出 `Inconclusive` 警告时，系统启动“专家介入机制”。

- **Broadcast Channel 广播频道**：系统会为该任务动态开辟一个只存活 24 小时的高性能频道。外部法医安全专家通过特定的验证 Token 进入控制台。
- **在线存在感知 (Presence)**：专家列表实时在面板亮起。
- **动态回注 (State Injection)**：专家不再是单纯看报告的看客，他们可以在关键证据帧处打下锚点并提交“人类人工分析论断”。这些论断会通过 WebSocket 以特殊的事件分发到后端。后端感知到 `expert_opinion` 事件后，会直接将其作为高权重的新节点插入到正在运行的 LangGraph State 缓存池中，强行扭转或干预 AI 的辩论进程。

## 7. 系统安全、数据合规与底层架构保障

作为针对国家级信息安全竞赛打造的作品，TruthSeeker 在底层架构与数据安全机制上实施了军工级的防线，确保分析结果具备法医级的不可篡改性与平台安全性。

### 7.1 联邦级隐私与防投毒保障 (PARSIFAL)

如第 3 节所述，底层的特征感知模型完全基于 PARSIFAL 架构训练。这不仅从物理上切断了敏感样本外泄的途径（只传输密文草图与符号梯度），更通过中位数聚合与密度估计，从数学原理上免疫了网络空间中恶意节点的后门投毒攻击，保障了上层 API 调用的绝对纯净。

### 7.2 数据合规与行级安全隔离（RLS）

系统全面利用了 PostgreSQL 的 Row Level Security (RLS) 策略。

- 明确的读写隔离：每一条任务数据、分析状态、原始素材都被打上严格的用户标识。用户仅能查阅自己提交的案件，而所有的推理中间态 (Analysis States) 对前端调用方实施绝对的**只读保护**，杜绝了越权操作的可能，保障了平台本身的数据资产安全。

### 7.3审计日志与报告防篡改机制

鉴于电子证据的法律严肃性，系统内置了完备的留痕与校验机制：

- **全程操作审计**：系统底层部署了 `audit_logs` 审计表，全量记录资源被上传、查看、下载及专家的介入操作。
- **Hash 校验锚点**：指挥官 Agent 最终生成的裁决报告 (`reports`) 在入库时，会自动基于内容特征提取生成全局唯一的 `report_hash`。这赋予了最终输出的法医证据链不可篡改的技术背书与法律效力。

### 7.4 向量级威胁储备仓库 (pgvector 赋能)

系统不仅能处理即时任务，更具有长期学习能力。通过引入 `pgvector` 扩展，系统将高频出现的欺诈特征沉淀为高维数据：

- **高危特征向量库 (threat_vectors)**：将 Deepfake 指纹、恶意域名等存入向量库，实现极速相似度比对。
- **Agent 记忆模型 (agent_memories)**：为后续大模型的微调与案例检索提供持久化的记忆支持，使得系统随着使用时间的增加而愈发敏锐。

## 8. UI/UX 创新：三维隐喻与呈现空间

### 8.1 抛弃扁平化的数字空间 (3D Bento Box)

在处理多模态跨维度的溯源案件时，传统的线性平铺列表会让安全人员认知过载。TruthSeeker 创造性地运用了 `@react-three/fiber` 呈现 3D Bento Box 控制台：

- 屏幕被物理划分为围绕中心原点悬浮的四大三维区块。视听鉴伪Agent流、源码情报、媒体原片、网络拓扑分别在 Z 轴上占据不同景深。
- 用户通过鼠标拖拽可以旋转整个观察视角。各个面板不再是死板的 `div`，而是存在于空间中的几何网格体。

### 8.2 材质着色核心：Liquid Glass (液体玻璃)

为了摒弃审美疲劳的 iOS 毛玻璃 (Frosted Glass)，系统在 Tailwind v4 层和 WebGL Shader 层创新出了一套 `Liquid Glass` 材质：

- 面板呈现一种具有流动感、不规则有机曲线包裹的半透明介质感。
- 结合物理级的光学折射效果 (`meshPhysicalMaterial with transmission`)。
- 当幕后对应 Agent 在进行大规模推断计算时，3D 卡片的边缘不仅会发起发光流转（Edge Glow），还会透射一种警示呼吸光晕（紫色为运算，红色为发现高危证据），给使用者极强的“情报正在汇聚”的生命感反馈。

## 9. 项目里程碑规划与演进路径

本项目采取严格的倒排期敏捷开发，划分为四层渐进式交付（Layer 1 到 Layer 4）。

- **M1 里程碑（核心鉴伪闭环）**：完成核心视听上传分析、搭建 Fast API 与基础的前端展示板。确立了底层的连通性。
- **M2 里程碑（全维度动态博弈）**：接入四位特化 Agent，最核心的收敛判定条件算法和动态路由打回逻辑完成联调跑通，系统证明其能自主完成无休止的推理提纯。
- **M3 里程碑（人机回环与 3D 展示）**：Supabase WebRTC/WebSocket 切入，人类专家可以在网页实时弹幕般干预 AI 断案。3D Liquid Glass 控制界面实装，大幅拉升项目外观体验维度。
- **M4 里程碑（实战对抗环境）**：配置演示内置案例库与“数据驾驶舱”。模拟在没有稳定公网网络甚至 API 提供商宕机的极端场馆环境下，利用数据缓存层（Redis）或 mock 系统让系统流畅丝滑路演。

## 10. 极限工程约束与核心踩坑经验

在落地该工业级架构的过程中，开发团队跨越了诸多前沿技术栈带来的生态震荡，形成了极为硬核的工程规范：

### 10.1 LangGraph v1.0+ 的状态机内存防线

后端的通信并不是简单的 JSON 满天飞，而是构建了不可变、带有 Reducer 特性的安全事务内存池 (`TruthSeekerState`)。

- **严格踩坑经验**：在早期开发中，团队曾使用 Pydantic BaseModel 作为状态载体，但导致了高并发下的运行时内存泄漏与类型解析瓶颈。团队最终严格贯彻 LangGraph v1.0+ 的 Breaking Changes 规范，**彻底封杀了 Pydantic 在 State 中的使用**。全面采用原生 `TypedDict` 与 `Annotated` 结合 `add_messages` 进行图节点流转编排，将状态机的稳定性提升至极高水准。

### 10.2 React 19 生态的前沿兼容挑战

为了追求极致的渲染性能，前端彻底切入了 Next.js 15 与 React 19 的新时代，但也面临了生态重构的巨大阻力：

- **组件库锁定**：抛弃了旧版 framer-motion，强制锁定全新的 `motion v12` 标准；为了兼容 React 19 的 3D 渲染，排雷后全面启用了处于 RC 阶段的 `@react-three/drei v10`。
- **样式引擎重构**：抛弃了传统的 Tailwind JS 配置流，全面采用 Tailwind v4 的 CSS-first 架构（通过 `@theme` 指令注入），大幅压降了构建体积与首屏时间。

### 10.3 极致的智能降级与回退控制器

为了应对恶劣网络环境与设备算力差异，系统设计了多级优雅降级策略：

- **3D WebGL 降级**：基于 Next.js RSC 机制，前端在初始化时探测硬件加速能力。若设备性能不足，底层无缝热切至极速的 CSS3 2D 回退模式，保证核心审判业务不受影响。
- **AI 模型备灾机制**：当主链路的跨模态大模型 API（如外部服务）出现高延迟或熔断时，系统能自动将流量切换至备用本地化策略（如自主部署的 Qwen3-VL 节点），确保办案流程永不卡死。

## 11. 核心实战场景与量化性能指标 (KPIs)

为了充分展示全模态、多智能体架构在复杂对抗环境下的压倒性优势，系统在实战演示环节预装了四大杀手级案例，并订立了严苛的性能检测标准。

### 11.1 四大复合型实战演示案例

系统并非停留在理论框架，而是能直击当前最具威胁的欺诈链路：

1. **“董事长语音诈骗”案**：输入为一段高仿语音留言。主要展示视听鉴伪Agent Agent 对音频克隆的频谱截断、合成伪影与声纹异常的高精度提取分析。
2. **“政要换脸视频”案**：主要展示在面对逐帧深层视频重绘时，系统的法医逻辑如何敏锐定位面部边缘的非自然融合以及光影相位差异。
3. **“钓鱼链接 + 伪造截图”案**：典型的跨模态混合场景。系统展示情报溯源 Agent 如何穿透视觉伪装，直捣恶意 URL 源头，进行域名黑名单碰撞与归属地溯源，从而推翻表象逻辑。
4. **“AI 生成新闻”案**：应对大语言模型生成的虚假报道。展示文本溯源追踪、AI 生成风格判定机制及图文逻辑的一致性比对。

### 11.2 核心量化指标基准 (KPIs)

TruthSeeker 通过了严格的工业级打磨，并在多维度的检测任务中确立了清晰的量化能力底线：

- **法医级精确度基准**：在多模态联合辩论策略的加持下，Deepfake 与复合型虚假信息的综合检测准确率目标稳定在 **> 90%**，同时将高敏感场景下的误报率严格压制在 **< 10%** 阈值内。
- **实战极限响应时效**：针对现场 Live 对抗演示要求——当测试者利用商业级 FaceSwap 应用现场生成假视频并上传时，系统从接收视频流、跨 Agent 并行查验到出具最终附带 Hash 校验的裁决报告，端到端完整目标耗时控制在 **< 90 秒**。系统以极致的工程效率，证明了其应对实时网络攻防战的高速运转能力。
