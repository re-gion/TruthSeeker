# 各大检测API讲解

## 视听鉴伪——Reality Defender API

**Reality Defender API** 是一个企业级的**多模态 Deepfake（深度伪造）和 AI 生成内容检测接口**。它的核心价值在于帮助开发者、安全团队、金融机构或通讯平台在真实生产环境中，防御由 AI 生成的合成身份欺诈、语音克隆诈骗以及虚假信息传播。

以下是关于该 API 输入、检测能力和输出的详细技术拆解：

### 1. 可以输入什么 (Inputs)

Reality Defender 是一个“API-First”的多模态平台，这意味着你可以通过统一的端点（Endpoint）处理多种类型的文件，而无需为不同的媒介调用不同的 API。

**支持的输入模态：**

- **视频 (Video):** `.mp4`, `.avi` 等动态视频流。
- **音频 (Audio):** `.mp3`, `.wav` 等语音片段。
- **图像 (Image):** `.jpg`, `.png` 等静态图片。
- **文本与文档 (Text/Documents):** 可疑的纯文本或文档文件。

**标准的上传工作流（REST API）：**

为了保证大文件的安全和高吞吐，API 采用 AWS 预签名 URL 的分步上传机制：

1. **鉴权：** 在 HTTP Header 中携带你的 `X-API-KEY`。
2. **获取上传签名：** 向 `/api/files/aws-presigned` 发起 `POST` 请求（携带文件名），获取一个临时的上传 URL。
3. **二进制直传：** 使用 `PUT` 请求，将文件的原始二进制数据直接推送到该签名 URL，无需附带其他额外的 Metadata。

### 2. 检测什么 (Detection Capabilities)

Reality Defender 并非依赖单一的二分类模型，而是采用了多模型集成（Ensemble）的方法，针对不同的生成技术进行交叉验证。

- **视觉伪造 (图像/视频)：** * 检测图像是否由扩散模型（如 Midjourney、Stable Diffusion）或 GANs 生成。
  - 检测视频中是否发生了**面部替换 (Face Swap)** 或**嘴唇同步篡改 (Lip-syncing)**。
  - 识别深度伪造算法在像素级或频域留下的伪影（Artifacts）。
- **声音伪造 (音频)：** * 检测音频是否由先进的 AI 语音合成或声音克隆工具（如 ElevenLabs）生成。
  - 识别合成语音中人耳难以察觉的声学不连贯性或环境底噪异常。
- **文本伪造：**
  - 分析文本模式，判断内容是否由大语言模型（LLMs）批量生成。

Reality Defender 作为一个多模态检测平台，能够直接解析标准视频文件（如 `.mp4`、`.mov` 等）中包含的音频轨道，并对音视频进行综合分析。

当你输入带音频的视频时，API 会在后台执行以下几项关键检测：

- **多模态联合分析 (Multimodal Analysis)：** 系统会分离视频流和音频流。它不仅会检测画面是否经过换脸 (Face Swap) 或 AI 生成，还会同步检测音频轨道中是否存在声音克隆 (Voice Cloning) 或 AI 合成语音 (TTS) 的痕迹。
- **唇音同步检测 (Lip-syncing Detection)：** 这是带音频视频最核心的检测维度之一。API 会交叉比对画面中人物的口型肌肉运动与音频的发音节点。如果是通过“音频驱动口型”生成的 Deepfake 视频（例如 Wav2Lip 技术），API 可以精准捕捉到音画在物理和声学上的微小不自然与不匹配。
- **细粒度的结果输出：** 在最终的 JSON 返回结果中，你不仅会得到一个总体的置信度得分，通常还能看到**独立的视频得分**和**独立的音频得分**。这能帮助你精确定位篡改发生在哪里——比如，视频画面可能是真实的，但音频是被 AI 篡改后重新配音的（或者反之）。

**总结来说：** 你不需要手动将视频和音频拆分开来提交。直接上传包含音频的原生视频文件，API 会自动完成拆解、多维度检测以及音画一致性的交叉验证。

### 3. 得到什么 (Outputs)

向 API 提交分析请求后，系统会在极短的推理时间（支持实时生产环境）内返回结构化的 JSON 数据。核心输出字段包括：

- **综合置信度得分 (Confidence Score)：** 一个直观的数值，代表该文件由 AI 生成的整体概率（例如 `0.98` 表示 98% 概率为伪造）。
- **二分类判定 (Binary Result)：** 直接给出 `Real`（真实）或 `Fake`（伪造）的结论，非常适合直接接入风控系统做自动拦截决策。
- **多维度可解释性指标 (Explainability Features)：** * **帧级别推断 (Frame-level Inference)：** 对于视频，API 会返回时间轴数据，精确指出视频中的**哪些特定帧**或**哪个时间段**被检测出伪造迹象，而不是笼统地否定整个视频。
  - **多模型独立得分：** 展示底层不同算法模型的独立判定结果，为安全分析师提供详实的取证（Forensic）依据。

### 💡 开发者生态扩展 (MCP 与微服务架构)

如果你正在使用现代化的 AI IDE 或构建基于 FastAPI 的多模态检测后端，Reality Defender 的生态提供了极大的便利：

- **官方 SDK 支持：** 提供 Python 和 Go 等主流语言的完整 SDK。
- **MCP (Model Context Protocol) Server：** 开源社区和官方已支持 Reality Defender 的 MCP Server。你可以直接将其挂载到你的 AI 编码助手中。MCP Server 内部甚至集成了一个轻量级的 FastAPI Web 服务，允许你通过暴露的 `GET /upload/{uuid}` 或 `POST /upload/{uuid}` 接口，直接在本地测试跨模态文件的上传与检测链路。



## 情报溯源——VirusTotal

**VirusTotal (VT) API** 是网络安全领域最权威的**恶意软件与威胁情报分析接口**。如果说 Reality Defender 是针对“AI 伪造内容”的安检机，那么 VirusTotal 就是针对“传统与新型网络威胁（如木马、钓鱼、勒索软件）”的超级聚合雷达。

它通过聚合全球 70 多家顶级防病毒引擎（如 Kaspersky, CrowdStrike, Microsoft Defender 等）和网站扫描器，为安全团队提供自动化分析和威胁追踪能力。

以下是关于 VirusTotal API（以其主流的 v3 版本为例）的详细技术拆解：

### 1. 可以输入什么 (Inputs)

VirusTotal 接受的输入范围非常广泛，几乎涵盖了网络攻防中所有常见的威胁指标（IoC - Indicators of Compromise）：

- **文件 (Files):** 可以上传任何类型的可疑文件（如 `.exe`, `.pdf`, `.apk`, `.docx`, 脚本文件等）。标准 API 支持最高 32MB 的文件直传，通过获取特殊上传 URL 可以支持更大文件（高达 650MB）。
- **哈希值 (Hashes):** 如果不想或不能上传文件实体，可以直接输入文件的 `SHA-256`, `SHA-1` 或 `MD5` 哈希值，查询该文件是否已经被全球其他安全研究员分析过。
- **网络标识符 (Network Indicators):**
  - **URL:** 具体的网页链接（用于检测是否为钓鱼网站或挂马网页）。
  - **域名 (Domains):** 例如 `suspicious-site.com`。
  - **IP 地址:** IPv4 或 IPv6 地址。

### 2. 检测什么 (Detection Capabilities)

VT 的检测不仅仅是简单的“杀毒软件扫描”，它是一个结合了静态分析、动态沙箱和关联分析的综合引擎：

- **多引擎签名交叉比对 (Multi-engine Scanning):** 将输入对象同时投喂给数十个防病毒引擎和 URL 扫描器，检测其是否命中已知的恶意软件特征库。
- **深度静态分析 (Static Analysis):**
  - 提取文件元数据：例如 PE 文件的导入/导出表、编译时间、数字签名证书是否被吊销。
  - 文档与脚本分析：提取 PDF 中的恶意 JavaScript，或 Office 文档中的恶意宏（Macros）。
  - **YARA 规则匹配：** 使用社区和官方的 YARA 规则扫描文件内容，发现隐藏的恶意代码片段。
- **动态沙箱行为分析 (Dynamic Sandbox Execution):** 将文件放入虚拟环境中（如 Windows, Android, macOS 沙箱）实际运行，监控其“越轨”行为，例如：
  - 是否修改了注册表？
  - 是否在后台悄悄释放了其他子文件（Dropped files）？
  - 是否发起了异常的网络外连请求？
- **网络信誉与基础设施图谱分析：** 检测一个 IP 或域名是否曾被用于僵尸网络（Botnet）控制节点 (C2)，或者是否与已知的黑客组织（APT）相关联。

### 3. 得到什么 (Outputs)

API 返回高度结构化且信息量极大的 JSON 数据。其核心输出包括：

- **检测统计 (Detection Stats):** 最直观的结果。例如 `{"malicious": 15, "suspicious": 2, "undetected": 54}`，这表明有 15 款引擎明确判定为恶意，2 款认为可疑。
- **分类与威胁标签 (Threat Labels):** 直接告诉你这是什么类型的威胁（如 `trojan.emotet`, `ransomware.wannacry`, `phishing`）。
- **行为指标 (Behavioral Indicators):** 沙箱运行后生成的动态报告，通常会映射到 **MITRE ATT&CK 框架**。例如，明确指出该文件使用了“T1055 进程注入”技术。
- **关系图谱 (Relationships):** VT 的强大之处在于其图谱能力。你可以得到：
  - 这个恶意 URL 曾经下载过哪些恶意文件实体？
  - 这个 IP 地址下面解析过哪些其他恶意域名？
  - 这个恶意软件在沙箱运行时，连接了哪些 IP？

### 💡 安全开发与项目集成

在构建安全自动化系统（例如**恶意 URL 检测系统**或威胁响应平台）时，VirusTotal API 是不可或缺的基础设施：

- **与本地模型的互补：** 如果你在后端（如 FastAPI）部署了自己训练的恶意 URL 深度学习检测模型（例如基于 CNN 或图神经网络），可以将 VirusTotal API 作为**外部真值源 (Ground Truth) 或兜底检测**。当本地模型置信度较低时，自动触发请求调用 VT API 进行云端研判。
- **自动化响应工作流 (SOAR)：** 结合 Python 和消息队列，可以实现：用户在前端提交一个可疑链接/文件 -> 后端调用 VT API 分析 -> 如果确认为高危威胁，直接联动防火墙或网关进行封堵。
