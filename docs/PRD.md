# TruthSeeker -产品需求文档 (PRD)

##1.项目背景与目标

###1.1背景
在生成式AI技术爆发式发展的2025-2026年，Deepfake伪造、AI换脸诈骗、多模态虚假信息传播已成为网络安全领域的核心威胁。传统单点检测方案面临以下挑战：
- **单一模型局限**：任何单一检测算法都存在盲区，易被针对性绕过
- **跨模态关联缺失**：视频、音频、文本的伪造往往存在内在矛盾，但现有工具难以联合分析
- **黑盒不可解释**：深度学习模型的"黑盒"特性导致检测结果难以被司法采信
- **溯源能力不足**：发现伪造后难以追踪信息来源和传播链路

###1.2目标
构建**TruthSeeker**——基于鲁棒联邦学习与多智能体交叉验证的跨模态鉴伪与溯源系统，冲击2026年全国大学生信息安全竞赛（CISCN）作品赛国家一等奖。

**核心差异化定位**：
> "不是另一个Deepfake检测工具，而是一个具备数字法医思维的智能研判中枢"

###1.3目标用户
- **普通网民**：快速鉴别收到的可疑视频/链接
- **平台审核员**：批量研判内容风险，生成标准化审核报告
- **安全研究人员**：深度分析新型伪造手法，积累威胁情报
- **执法人员**：获取具有证据效力的技术分析报告

##2.核心业务链路

###2.1主流程：单次检测任务

```
用户上传内容 →模态识别与优先级标记 →并行预处理
 ↓
 ┌───────────────┼───────────────┐
 ↓ ↓ ↓
 [视听鉴伪Agent] [情报溯源Agent] [其他模态Agent]
 │ │ │
 └───────────────┼───────────────┘
 ↓
 [全局证据板 State]
 ↓
 [逻辑质询Agent审视]
 ↓
 ┌───────────────┴───────────────┐
 ↓ ↓
 [收敛?终止] [未收敛?继续辩论]
 ↓ │
 [研判指挥Agent裁决] ←─────────────────────┘
 ↓
 [生成法医级报告]
 ↓
 [数据库存档 +导出PDF/MD]
```

###2.2子流程：专家会诊模式

```
辩论进行中 →触发人工介入条件 →暂停自动流程
 ↓
邀请外部专家加入Realtime频道
 ↓
专家查看当前证据板 +历史轮次
 ↓
专家提交质疑/确认/补充证据
 ↓
作为新节点注入State →恢复自动辩论
```

##3.功能模块定义

###3.1前端应用层

|模块 |功能描述 |优先级 |
|------|----------|--------|
| **3D Bento控制台** |四象限空间布局（原媒体（上传内容展示）/视听法医流/溯源图/质询日志），支持3D旋转切换视角 | P0 |
| **智能上传器** |支持拖拽上传视频/音频/图片/文本，自动识别模态类型，用户可标记优先级 | P0 |
| **实时状态墙** |展示各Agent当前状态（思考中/调用API/等待响应），Liquid Glass呼吸光效 | P0 |
| **证据时间轴** |垂直时间轴展示辩论轮次，可展开查看任意时刻的完整State快照 | P1 |
| **专家会诊面板** |实时协作界面，支持发表评论、标记疑点 | P2 |
| **报告展示页** |沉浸式Scroll-telling呈现最终报告，支持3D翻转查看详情 | P0 |
| **案例库** |内置典型攻击案例，一键体验完整分析流程（竞赛演示用） | P1 |
| **数据大屏** |系统运行统计数据可视化（总量/威胁分布/响应时间） | P1 |

###3.2后端服务层

|模块 |功能描述 |优先级 |
|------|----------|--------|
| **LangGraph引擎** |四Agent状态机编排，动态权重收敛判定，最大轮数限制 | P0 |
| **视听鉴伪Agent** |视频抽帧、音频分离、调用视觉/音频检测API、量化指标提取 | P0 |
| **情报溯源Agent** | URL/IP提取、Whois查询、恶意域名检测、OSINT情报聚合 | P0 |
| **逻辑质询Agent** |审视证据板、生成质疑Prompt、设定条件边推动深入检测 | P0 |
| **研判指挥Agent** |终局裁决、置信度加权、报告生成、工作流终止 | P0 |
| **Realtime广播服务** | Supabase Realtime集成，State变更推送、专家协作消息 | P0 |
| **报告生成器** | Markdown/PDF导出、SHA-256完整性哈希 | P1 |
| **智能降级控制器** | API故障检测、自动切换备用策略、优雅降级 | P1 |

###3.3外部能力层（Tools）

| Tool类别 |候选服务 |用途 |
|----------|----------|------|
| **视觉伪造检测** | Microsoft Video Authenticator / Reality Defender |面部篡改检测、帧间一致性分析 |
| **音频伪造检测** | Resemble Detect / ElevenLabs检测API /自建Whisper+Classifier |语音克隆检测、频谱异常分析 |
| **恶意URL识别** | VirusTotal API / Google Safe Browsing / URLVoid |域名信誉查询、威胁情报关联 |
| **开源情报查询** | Shodan / Censys / WHOIS XML API | IP归属、域名注册信息、资产测绘 |
| **多模态大模型** | Kimi/Moonshot (moonshot-v1-8k) |伪装专家分析、跨模态关联推理 |

##4.非功能性需求

###4.1性能指标
-短视频（<30s）：端到端检测 <60秒
-长视频（<10min）：端到端检测 <5分钟
-前端首屏加载 <2秒（3D资源懒加载）
- Realtime延迟 <500ms

###4.2可靠性
-服务可用性 >99%（竞赛演示期间）
-检测任务失败率 <5%
- API故障时自动降级成功率 >95%

###4.3安全性
-用户上传文件端到端加密存储
-原始文件7天后自动删除（可配置）
-完整的操作审计日志
- RLS行级安全策略隔离用户数据

###4.4可扩展性
-支持水平扩展Worker处理队列
-新Agent类型可插拔注册
-新Tool API可配置化接入

##7.风险与应对

|风险 |可能性 |影响 |应对措施 |
|------|--------|------|----------|
|外部API不稳定 |高 |高 |实现智能降级，备用大模型方案 |
