# TruthSeeker -产品需求文档 (PRD)

##1.项目背景与目标

###1.1背景
在生成式AI技术爆发式发展的2025-2026年，Deepfake伪造、AI换脸诈骗、多模态虚假信息传播已成为网络安全领域的核心威胁。传统单点检测方案面临以下挑战：
- **单一模型局限**：任何单一检测算法都存在盲区，易被针对性绕过
- **跨模态关联缺失**：视频、音频、文本的伪造往往存在内在矛盾，但现有工具难以联合分析
- **黑盒不可解释**：深度学习模型的"黑盒"特性导致检测结果难以被司法采信
- **溯源能力不足**：发现伪造后难以追踪信息来源和传播链路

###1.2目标
构建**TruthSeeker**——面向跨模态鉴伪与情报溯源的 FedPaRS-compatible 多智能体研判系统，冲击2026年全国大学生信息安全竞赛（CISCN）作品赛国家一等奖。

**核心差异化定位**：
> "不是另一个Deepfake检测工具，而是一个具备数字法医思维的智能研判中枢"

###1.3目标用户
- **普通网民**：快速鉴别收到的可疑视频/链接
- **平台审核员**：批量研判内容风险，生成标准化审核报告
- **安全研究人员**：深度分析新型伪造手法，积累威胁情报
- **执法人员**：获取具有证据效力的技术分析报告

##2.核心业务链路

###2.1主流程：单次检测任务

```
用户上传内容 + case_prompt
 ↓
[电子取证Agent: Kimi2.6全模态理解 + RD/VT all-settled工具检测]
 ↓
[逻辑质询Agent审查取证报告，按质量阈值收敛或打回]
 ↓
[情报溯源Agent: Exa脱敏搜索 + VT追加查询 + 溯源图谱生成]
 ↓
[逻辑质询Agent审查图谱节点、边、引用和模型推断关系]
 ↓
[研判指挥Agent生成最终鉴伪与溯源报告]
 ↓
[逻辑质询Agent审阅报告，必要时只打回Commander修订]
 ↓
[数据库存档 + 图谱视图 + 导出PDF/MD + report_hash]
```

###2.2子流程：专家会诊模式

```
辩论进行中 →触发人工介入条件 →暂停自动流程
 ↓
邀请外部专家加入Realtime频道
 ↓
专家查看当前证据板 +历史轮次
 ↓
专家提交质疑/确认/补充证据
 ↓
作为新节点注入State →恢复自动辩论
```

##3.功能模块定义

###3.1前端应用层

|模块 |功能描述 |优先级 |
|------|----------|--------|
| **3D Bento控制台** |四象限空间布局（原媒体展示/电子取证流/溯源图/质询日志），支持3D旋转切换视角 | P0 |
| **智能上传器** |支持拖拽上传视频/音频/图片/文本，自动识别模态类型，用户可标记优先级 | P0 |
| **实时状态墙** |展示各Agent当前状态（思考中/调用API/等待响应），Liquid Glass呼吸光效 | P0 |
| **证据时间轴** |垂直时间轴展示辩论轮次，可展开查看任意时刻的完整State快照 | P1 |
| **情报溯源图谱视图** |展示最终审定 provenance graph，支持节点筛选、边筛选、详情和引用面板；当前内置 SVG 组件仅用于验证数据链路，最终交付仍要求 `@xyflow/react` / React Flow 交互画布 | P0 |
| **专家会诊面板** |实时协作界面，支持发表评论、标记疑点 | P2 |
| **报告展示页** |沉浸式Scroll-telling呈现最终报告，支持3D翻转查看详情 | P0 |
| **案例库** |内置典型攻击案例，一键体验完整分析流程（竞赛演示用） | P1 |
| **数据大屏** |系统运行统计数据可视化（总量/威胁分布/响应时间） | P1 |

###3.2后端服务层

|模块 |功能描述 |优先级 |
|------|----------|--------|
| **LangGraph引擎** |四Agent状态机编排，动态权重收敛判定，最大轮数限制 | P0 |
| **电子取证Agent** |保留 `forensics` 协议 key，接收全模态输入，调用 Reality Defender 和 VirusTotal，生成取证报告 | P0 |
| **情报溯源Agent** |基于 Exa、VirusTotal、样本线索和取证输出生成 OSINT 溯源图谱 | P0 |
| **逻辑质询Agent** |分阶段审查取证报告、溯源图谱和最终报告，控制收敛与打回 | P0 |
| **研判指挥Agent** |综合全局证据板、辩论过程和图谱生成最终鉴伪与溯源报告 | P0 |
| **Realtime广播服务** | Supabase Realtime集成，State变更推送、专家协作消息 | P0 |
| **报告生成器** | Markdown/PDF导出、SHA-256完整性哈希 | P1 |
| **智能降级控制器** | API故障检测、自动切换备用策略、优雅降级 | P1 |

###3.3外部能力层（Tools）

| Tool类别 |候选服务 |用途 |
|----------|----------|------|
| **视觉伪造检测** | Microsoft Video Authenticator / Reality Defender |面部篡改检测、帧间一致性分析 |
| **音频伪造检测** | Resemble Detect / ElevenLabs检测API /自建Whisper+Classifier |语音克隆检测、频谱异常分析 |
| **恶意URL识别** | VirusTotal API / Google Safe Browsing / URLVoid |域名信誉查询、威胁情报关联 |
| **开源情报查询** | Exa API / Shodan / Censys / WHOIS XML API | 脱敏联网搜索、IP归属、域名注册信息、资产测绘 |
| **多模态大模型** | Kimi/Moonshot (kimi-k2.6) |四 Agent 共享的多模态基本认知、跨模态推理和报告生成 |

##4.非功能性需求

###4.1性能指标
-短视频（<30s）：端到端检测目标 <5分钟，偏质量优先
-长视频（<10min）：端到端检测目标 <10分钟，工具超时后结构化降级
-前端首屏加载 <2秒（3D资源懒加载）
- Realtime延迟 <500ms

###4.2可靠性
-服务可用性 >99%（竞赛演示期间）
-检测任务失败率 <5%
- API故障时自动降级成功率 >95%

###4.3安全性
-用户上传文件端到端加密存储
-原始文件7天后自动删除（可配置）
-完整的操作审计日志
- RLS行级安全策略隔离用户数据

###4.4可扩展性
-支持水平扩展Worker处理队列
-新Agent类型可插拔注册
-新Tool API可配置化接入

##7.风险与应对

|风险 |可能性 |影响 |应对措施 |
|------|--------|------|----------|
|外部API不稳定 |高 |高 |实现智能降级，备用大模型方案 |
|3D性能问题 |中 |中 |提供2D fallback模式，优化LOD |
|检测精度不达预期 |中 |极高 |保留 FedPaRS-compatible 检测器替换接口，准备对比实验数据 |
|开发进度延期 |中 |高 |严格分层交付，确保MVP先可用 |
