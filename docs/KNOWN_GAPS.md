# TruthSeeker 已知缺口

> 审计日期：2026-06-06。本文只记录已由代码或迁移静态审计确认、但本轮文档整理未擅自修改的高风险事项。完成修复后应补测试并从本清单移除。

## P0：上线前必须处理

| 缺口 | 当前证据 | 影响 | 建议验收 |
| --- | --- | --- | --- |
| 公开案例与公开案例 RAG 的写策略过宽 | 公开案例相关迁移允许任意 authenticated 用户写入或更新共享数据 | 登录用户可能篡改公共案例或向量块 | 收紧到服务端受控写入；用不同账号执行 RLS 集成测试 |
| canonical `/collaboration` 专家路径未纳入生产公开路由判断 | 路由已挂载，但认证中间件的邀请/专家访问白名单仍只覆盖 `/consultation` | 启用生产 JWT 后，新专家链接流程可能被认证拦截 | 补 canonical 路径认证测试，并验证邀请查看、发言、历史读取 |
| 从零部署缺少 `media` Storage bucket 与 Storage RLS 迁移 | 上传接口固定写入 `media`，现有 SQL migrations 未创建 bucket/policy | 新 Supabase 项目不能只依赖仓库迁移跑通上传 | 增加幂等 Storage 迁移，在空项目完成真实上传/读取验证 |
| 新协作迁移可能被旧空 `expires_at` 阻断 | 新邀请表要求 NOT NULL，复制旧邀请时未提供空值兜底 | 有历史空值的数据库可能迁移失败 | 用含空过期时间的旧数据执行迁移回归测试 |

| 缺口 | 影响 | 建议 |
