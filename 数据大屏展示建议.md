## 二、这些数据怎么放进你的大屏，逻辑才顺

我建议你把大屏拆成 **三层逻辑**，而不是按“报告一块、报告一块”去堆。

第一层叫“**风险底座**”。这里放 CNNIC 和网络视听报告的数据，回答的是：现在生成式 AI 和视听内容已经普及到什么程度，为什么 Deepfake 风险会爆发。最适合放“2.49 亿生成式 AI 用户”“10.91 亿网络视听用户”“10.40 亿短视频用户”“近三分之一网民用 AI 做图视频”这些数字。[cnnic.net.cn+2中国服务贸易指南网+2](https://www.cnnic.net.cn/NMediaFile/2025/0220/MAIN1740036167004CKE0DITFO1.pdf)

第二层叫“**治理环境**”。这里放国家网信办备案公告、国家信息化发展报告、网络法治发展报告的数据，回答的是：国家为什么要治理、治理已经推进到什么程度。最适合放“302 款备案、238 款新增、105 款登记”“注册用户超过 6 亿、头部日活破 3000 万”“11159 家约谈、4046 家处罚、10946 家违法网站关闭”这些数据。[国家管理局+2国家管理局+2](https://www.cac.gov.cn/2025-01/08/c_1738034725920930.htm)

第三层才叫“**系统能力层**”。这里放你自己的真实数据：累计检测任务数、威胁类型分布、平均响应时间、活跃 Agent 数，以及后续你可以加的“模态来源占比、专家介入次数、白盒报告生成数、来源溯查命中数”。这样评委会形成非常自然的因果链：**外部风险大 → 国家治理强 → 你的系统正好解决这个场景中的关键缺口**。

## 三、去热门组件库里找，哪些图最适合你

### A. 主推荐：Apache ECharts

如果你想要“**竞赛最稳 + 可控 + 适合大屏**”，首选还是 ECharts。官方明确说明它是开源 JavaScript 可视化库，内置 **20 多种图表类型**，并且带有十余种组件，可自由组合；常用系列里就包括 **Bar、Line、Pie、Radar、Heatmap、Graph、Gauge、Sankey、ThemeRiver**，组件里还有 **Timeline、DataZoom、VisualMap、Toolbox**。这对你的数据大屏非常合适。[echarts.apache.org+1](https://echarts.apache.org/)

我建议你在 ECharts 里重点用这几种：

1. **Bar / Horizontal Bar**  
    适合：短视频、微短剧、直播、音频等不同内容形态对比；也适合“备案数 / 新增数 / 登记数”“约谈 / 处罚 / 关站”等治理数据对比。  
    原因：这类数据本质是类别比较，柱状图最稳，评委一眼就能看懂。ECharts 官方支持 Bar。[echarts.apache.org](https://echarts.apache.org/v4/en/cheat-sheet.html)
2. **Area Chart**  
    适合：平均响应时间趋势、每日检测任务趋势、近 7 天活跃 Agent 波动。  
    原因：面积图比普通折线更有“趋势场”的感觉，官方也明确说明它在少量序列场景下比普通折线更直观。[echarts.apache.org](https://echarts.apache.org/handbook/en/how-to/chart-types/line/area-line/?utm_source=chatgpt.com)
3. **Rose Chart（Nightingale Chart）**  
    适合：威胁类型分布、攻击载体分布、模态占比。  
    原因：ECharts 官方把它定义为 Nightingale Chart，用相同角度、不同半径表达类别，非常适合大屏“既有数值比较，又要视觉冲击”的场景。你这类安全大屏拿它做“威胁类型占比”，会比普通饼图更有记忆点。[echarts.apache.org](https://echarts.apache.org/handbook/en/how-to/chart-types/pie/rose/)
4. **Sankey**  
    适合：来源 → 模态 → 证据 → 最终裁决 的流向展示；也适合 “上传内容 → 各 Agent → 结论” 的系统流程展示。  
    原因：Sankey 天然适合展示“流”和“权重”，很符合你项目“证据链式分析”的核心气质。ECharts 官方支持 Sankey。[echarts.apache.org](https://echarts.apache.org/v4/en/cheat-sheet.html)
5. **Graph**  
    适合：账号、域名、IP、平台、传播节点之间的关系网；来源溯查结果。  
    原因：你的系统有 URL / WHOIS / 威胁情报 / 来源溯查模块，用 Graph 做关系网络比静态列表更像“数字法医”。ECharts 官方支持 Graph。[echarts.apache.org](https://echarts.apache.org/v4/en/cheat-sheet.html)
6. **Gauge**  
    适合：今日活跃 Agent 数、系统健康度、综合风险分、置信度。  
    原因：仪表盘类组件在状态类指标上非常直观，特别适合大屏角落放“运行态”。ECharts 官方支持 Gauge。[echarts.apache.org](https://echarts.apache.org/v4/en/cheat-sheet.html)

### B. React 场景很顺手：Ant Design Charts（AntV）

如果你更看重“**在 React 前端里快速落地**”，Ant Design Charts 很适合。它官方直接把自己描述为“**Simple and easy to use React chart library**”，并强调“几行代码就能生成高质量统计图”，同时覆盖统计图、关系图、流程图、地理可视化。[ant-design-charts.antgroup.com](https://ant-design-charts.antgroup.com/en)

对你这个项目，我最建议看的几个官方图表名是：

1. **DualAxes / Line Bar Chart**  
    适合：市场规模 + 用户规模、备案总数 + 年度新增、任务总量 + 平均响应时间。  
    原因：你有很多“一组绝对值 + 一组趋势值”的数据，双轴图比单图更节省屏幕。Ant Design Charts 官方组件里就有 DualAxes，示例里包含 **Line Bar Chart**、**Multi Axis Chart**。[ant-design-charts.antgroup.com+1](https://ant-design-charts.antgroup.com/en/examples?utm_source=chatgpt.com)
2. **Nightingale Rose Chart**  
    适合：威胁类型分布、模态来源分布。  
    原因：它比普通 Pie 更适合大屏风格化表达，官方示例里直接提供 **Nightingale Rose Chart**。[ant-design-charts.antgroup.com](https://ant-design-charts.antgroup.com/en/examples)
3. **Liquid Chart**  
    适合：活跃 Agent 占比、系统资源利用率、审核完成率。  
    原因：水球图很“仪表化”，适合放在控制台小区域做实时态展示。官方示例里有 **Liquid Chart**、**Background Liquid Chart**、**Pin Liquid Chart**。[ant-design-charts.antgroup.com](https://ant-design-charts.antgroup.com/en/examples)
4. **Gauge Chart**  
    适合：综合风险分、可信度评分、实时服务健康状态。  
    原因：安全系统大屏里很常用，官方示例里直接有 **Gauge Chart**。[ant-design-charts.antgroup.com](https://ant-design-charts.antgroup.com/en/examples)
5. **Sankey**  
    适合：证据链流向、Agent 协作流。  
    原因：你的系统流程和证据流都很适合流向图，官方示例有 **Alipay sankey**、**Custom node Color** 等。[ant-design-charts.antgroup.com](https://ant-design-charts.antgroup.com/en/examples)
6. **NetworkGraph / FlowGraph**  
    适合：溯源关系网络、传播链路。  
    原因：你项目本来就有来源溯查和威胁情报联动，关系图会比简单表格更像“研判系统”而不是“后台管理页”。Ant Design Charts 的组件总览里包含 **NetworkGraph**、**FlowGraph**、**FlowDirectionGraph**。[ant-design-charts.antgroup.com](https://ant-design-charts.antgroup.com/en/components/plots/dual-axes)

### C. 高质感备选：Highcharts

Highcharts 的优点是观感成熟、交互细腻、官方 demo 很适合做高完成度展示。官方说明 Highcharts Core 有 **40+ chart types**，并支持 React；同时官方也明确写了，**生产和商业使用需要商业授权**。如果你目前只是比赛演示或非商用验证，可以考虑；如果后面真要产品化，最好提前评估授权成本。[Highcharts+1](https://www.highcharts.com/download/)

如果你选 Highcharts，我建议只看三个：

1. **Sankey diagram**  
    适合：证据流、传播流、模态流。  
    原因：官方直接说明 Sankey 用来可视化节点之间的数据流和流量，线越宽表示量越大，这和你的“证据链”概念高度一致。[Highcharts](https://www.highcharts.com/demo/sankey-diagram?utm_source=chatgpt.com)
2. **Solid gauge**  
    适合：实时风险值、活跃 Agent、置信度。  
    原因：官方说明 `solidgauge` 是 gauge 的实心版本，更适合仪表化大屏状态。[Highcharts](https://www.highcharts.com/docs/chart-and-series-types/angular-gauges)

这是你和普通“检测器”最大的差异点。你的项目报告强调的恰恰是“证据板、逻辑质询、收敛判定、专家回注、白盒报告”，而不是只输出一个真假分值。用 Sankey，评委能一眼看见“系统在推理”。

### 5）模块：来源溯查关系图

建议图表：**Graph / NetworkGraph**
推荐组件来源：ECharts 官方 `Cheat Sheet` 明确支持 `Graph`；Ant Design Charts 的组件概览里也有 `NetworkGraph`、`FlowGraph` 等关系图能力。([ECharts](https://echarts.apache.org/en/cheat-sheet.html?utm_source=chatgpt.com))

建议字段口径：

- `domain`
- `ip`
- `platform`
- `account`
- `case_id`
- `threat_intel_tag`

为什么这样展示：
你的系统不只是查内容真假，还要做 URL、WHOIS、威胁情报和来源线索联动。关系图能把“溯源”二字真正可视化。

### 6）模块：专家会诊与白盒报告

建议图表：**Gauge + Horizontal Bar**
推荐组件来源：ECharts `Gauge`；Ant Design Charts 官方组件里也有 `Gauge`、`Liquid`。([ECharts](https://echarts.apache.org/en/cheat-sheet.html?utm_source=chatgpt.com))

建议字段口径：

- `expert_interventions_total`
- `expert_confirmed_cases`
- `expert_reopened_cases`
- `report_export_total`
- `report_pdf_total`
- `hash_verified_total`

为什么这样展示：
这块是你的“可信研判”收口。项目报告已明确系统输出白盒报告，包含关键证据链分析、跨模态矛盾、来源溯查结果、人工介入记录、最终裁决与哈希校验锚点。大屏里把“专家会诊次数”和“白盒报告生成数”拉出来，会很加分。

## 四、我建议你最终做成的页面结构

最稳的布局不是平均分三层，而是：

上层 25%：**风险底座层**
中层 20%：**治理环境层**
下层 55%：**系统能力层**

原因很简单：前两层负责“立题”，第三层负责“打分”。评委会因为前两层认可这个项目有现实意义，但最终给高分，还是因为第三层能证明你的系统不是 PPT。这个权重分配，也更符合你 PRD 里“数据大屏是系统运行统计可视化”的原始定位。

## 五、库的选择，我给你一个明确结论

**首选 ECharts，局部借鉴 Ant Design Charts，Highcharts 只做备选。**

原因：

- ECharts 开源，官方明确是开源 JavaScript 可视化库，支持 20 多种图表和十余种组件组合，适合大屏与安全态势风格。([ECharts](https://echarts.apache.org/?utm_source=chatgpt.com))
- Ant Design Charts 对 React 更友好，官方主打“开箱即用”，而且 `DualAxes`、`Gauge`、`Liquid`、`Sankey` 这些你会用得上。([Ant Design Charts](https://ant-design-charts.antgroup.com/?utm_source=chatgpt.com))
- Highcharts 质感成熟，也有 React 官方集成和 40+ chart types，但官方明确写了**生产和商业使用需要商业授权**，更适合后续产品化再考虑。([highcharts.com](https://www.highcharts.com/download/?utm_source=chatgpt.com))
